# На первом митапе
Рассказать об области применения SE2 и варианты его использования в терминах бизнес-логики:
- был разработан, как средство автоматизации процессов производства сложных приборов.
- разные этапы производства: от диодов до готовых приборов/ Экскурсия: показать приборы, блоки, кабинеты.
- разные специалисты, использующие автоматизацию, соотвественно разные сценарии использования (разработка, производство, метрология)
- скрипт - это последовательность определенных действий (чаще всего измерений и их анализ), связанных с проверкой или настройкой тестируемого устройства (в широком смысле).
- SE2 имеет гибкую модульную структуру скрипта, которая позволяет писать сложные сценарии.

# Модули SE2:
## Для работы с приборами
- любые приборы, поддерживающие SCPI/MICS-команды
- генератор / скалярный анализатор (Г7М, Р2М)
- анализатор спектра / измеритель коэффициента шума (СК4М, Х5М)
- измеритель мощности M3M
- генератор сигналов PLG
- умный мультиметр Appa
- термокамера
## Для отображения информации в GUI
- График
- Запрос входных данных
- Список
- Таблица
- Лог
- Индикатор прогресса
- Параметры калибровки
## Вспомогательные
- Работа по ftp
- Работа с базой данных
- Реализация метода наименьших квадратов
- Работа с таблицами Excel

# Рассказать о решаемой проблеме в терминах бизнес-логики
- скрипты разрабатываем и поддерживаем мы - на основе запросов производства, разработчиков, метрологов.
- самовольное изменение скриптов на производстве (проблема отслеживания).
- скриптов много, разработчиков много
- доставка актуальных скриптов всем пользователям (не всегда известны все пользователи скриптов).
- отсуствие единого хранилища скриптов для хранения актуальных версий.
- желательно иметь возможность быстро развернуть необходимые скрипты для того или иного прибора.
