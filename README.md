# script_storage

## База

- Всеми файлами теперь обменивамся через [репозиторий](https://github.com/MicranDIIS/script_storage), чтобы материалы были в одном месте, не терялись и были доступны всем.

- Обсуждения по технической части проекта ведем в [дискуссиях](https://github.com/MicranDIIS/script_storage/discussions) или в карточках задач (см. [ниже](##Задачи)). Помним, что в обсуждениях можно тагать пользователей.

- **Не боимся коммитить** свои наработки! Коммиты, какими бы они не были (вас никто не осудит :) ), показывают ход вашей работы, ход ваших мыслей. Это позволит вам быть в курсе работы друг друга, не потерять важное, а нам вас координировать и давать обратную связь.

## Митапы

- Материалы по митапам загружаем в папку [meetups](https://github.com/MicranDIIS/script_storage/tree/main/meetups), в подпапку, которая соответствует  дате митапа. 

## Регламент разработки

### Задачи

- Задачи для решения берем из [проекта](https://github.com/orgs/MicranDIIS/projects/3/views/1) - передвигаем их из одной колонки в другую, в соответствии со статусом.

- Также список задач можно найти в [репозитории](https://github.com/MicranDIIS/script_storage) в разделе [issues](https://github.com/MicranDIIS/script_storage/issues). 

- Если по задаче есть вопросы или предложения, то заходим в карточку задачи ([пример](https://github.com/MicranDIIS/script_storage/issues/18)), пишем снизу комментарий и нажимаем **Comment**.

- Задачи имеют приоритет - сначала решаем задачи с более высоким приоритетом, а лишь потом с более низким.

- Приоритет у задач может меняться. 

- Задачи назначаем мы, либо вы сами можете назначить себя на определенную задачу, выбрав себя в поле **Assignees** в карточке задачи.

### Ветки
Для поддержания порядка в проекте придерживаемся следующих правил работы с ветками:

- для каждой задачи создается отдельная ветка (пока создавать эти ветки можно от ваших текущих веток, не обязательно от main)
- имя ветки должно соответствовать формату: <тип>/<номер_задачи>-<краткое_описание>
- <тип> - может быть одним из представленных ниже:
  - feat/ — новая функционал
  - fix/ — исправление багов
  - refactor/ — изменение кода без смены логики (чистка кода)
  - style/ — правки форматирования (пробелы, отступы, не влияющие на код)
  - perf/ — правки для повышения производительности
  - docs/ — правки в документации
  - test/ — добавление или изменение тестов
  - chore/ — обновление зависимостей, настроек сборки и прочие рутинные задачи
        
- имя ветки должно быть на латинице

#### Пример
Для задачи ["базовая сортировка по дате #44"](https://github.com/MicranDIIS/script_storage/issues/44) имя ветки будет **feat/44-basic-date-sorting**
